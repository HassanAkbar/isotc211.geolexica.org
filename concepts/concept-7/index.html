Term ID: 7
Definition: process of determining whether an implementation satisfies acceptance criteria and enables the user to determine whether to accept the implementation
notes: Note 1 to entry: This includes the planning and execution of several kinds of tests (e.g. functional, volume, performance tests) that demonstrate that the implementation satisfies the user requirements.Note 2 to entry: This is not a part of conformance testing

Term ID: 7

Language: jpn

Term: 受入試験
Term ID: 7
Definition: ある実装が利用者の受入基準を満たすかどうかを決定し，利用者がそれを受け入れるかどうかを判断するための試験。
notes: 備考1. 受入試験には，実装が利用者の要求を満たしていることを立証するための，種々の試験（機能性試験，数量試験，性能試験など）の計画及び実行が含まれる。備考2. 受入試験は，適合性試験の一部ではない。
